两数之和

    给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那两个整数，并返回他们的数组下标。 你可以假设每种输入只会对应一个答案。但是，你不能重复利用这个数组中同样的元素。

示例

给定 nums = [2, 7, 11, 15], target = 9 因为nums[0] + nums[1] = 2 + 7 = 9,所以返回[0, 1]

思路

  • 最容易想到的就是暴力法，更快的就是Hash表，也可以利用数组实现。
  • 需要注意的是，暴力法中循环的下标和数组实现过程中对于负数的处理。

    • 暴力法
    • Hash表
    • 数组

    注意Hash表的创建以及对其的操作。

    • Class HashMap<K,V>
      • K:keys(键值)
      • V:Value(键值对应的数值)
      • 传递的参数是Integer，Integer是int的封装类，两者区别很大。
    • boolean containsKey(Object key)
          如果表中包含键值则返回true。
    • V put(K key, V value)
          创建hash表的映射关系。因为题目要求返回下标，所以可以建立nums[i]→i的映射。

    C++解法

    在C++中，通过 hashtable作为底层实现的主要有两个容器：

    • unordered_set，一个单类型数据的 set结构，内部元素不能重复，比如 name_set={"Mike", "John"}
    • unordered_map，一个 key-value键值对型的容器，比如 id和 name类型的 map：info_map={1:"Mike", 2:"John"}。

    而在unordered_map中， find用于进行关键字的查询，找到则指向关键字的迭代器，否则返回指向 end的迭代器。在上例中， it就是迭代器， it->first指向 key，而 it->second指向 value。

    int型数组转为二进制的函数 void toBin(int *key, int *keyin)，具体如下：

    进行两次调用，结果第二次输出全部为 0。

    问题出在 toBin函数中，传递进去的是两个指针，第一次传入的 plaintext存放的数，被每次 /2直到全部为0了，所以要么重新传递进去一个 plaintext，要么在 toBin中不直接用 keyin操作，而是先将 keyin转移到另一个数组再进行操作。

    字符数组

    创建字符数组：char* s = "This is a test"，如果要对每8位进行操作，则可以使用如下操作：

    两个需要注意的点：

    • 字符数组以 ‘\0’为结束标志，ASCII码为0，所以在for循环中可以直接用 s[i]作为结束标志。
    • s + i不是表示增加ASCII码，char* s是一个指针，指针加上一个数表示将指针移动相应的位置，比如 i = 8时，s的位置就移动到了 a的位置。

    字符转二进制：

    src是要转换的字符数组，比如 src = “romantic”，长度 length = 8，bit是存放转换结果的字符数组。依次取出字符，比如现在 t = ‘r’，而 ‘r'的 ASCII码为 114，二进制表示为 01110010。那么 t & 1就取出了末位的 0，然后 t >>= 1，就得到的倒数第二位的 1，依次就能全部变为二进制。存放在 bit应该是连续的，i << 3 + j刚好就使得下标得以连续，相当于 i * 8 + j。

    二进制转字符：

        跟上面的想对应，长度为8的字符数组对应64位二进制数。所以这里循环次数是 length << 3，扩大8倍，每8个二进制数对应一个字符，因此 str下标为 i >> 3，首先将 str[i >> 3] <<= 1左移1位，那么无论本来末位是0还是1，都会变成0，再与 bin[i + 1]相或就成功添加到了末位，然后再左移，重复这个过程。 至于为什么是 bin[i + 1],是因为上面字符转二进制时，j = 8; j > 0，所以下标是从1开始的。

    参数传递问题

        在传递参数时，函数本来应该是 char* offset，但是却传进去一个 int型的数组，这样在对 offset遍历时会出现一个什么问题呢？

    • 除了 offset[0]是对的以外，其余的 offset[i]都是 0('\0'的ASCII码)。应该是转换成 char*的时候，自动在第一个元素后面补了一个结束标志，即 '\0'。
    • 所以说参数的类型一定要搞清楚。
