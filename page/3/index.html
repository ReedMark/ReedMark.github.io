计算机网络-数据链路层

Posted on 2020-04-11 Edited on 2020-04-14 In 计算机网络
链路控制
主要功能
  1. 封装成帧：在一段数据的前后分别添加首部和尾部，接收方可以根据首部和尾部的标记，识别帧的开始和结束。
  2. 透明传输：上层交下来的数据，不管是什么形式的比特组合，都必须能够正确传送。
  3. 差错检测：实际通信链路不会是理想的，为了保证数据传输的可靠性，需要在数据链路层进行差错检测。
数据链路层做成可靠的链路层的优劣
  1. 优点：使网络中的某个结点及早发现传输中出了差错，然后经过重传来纠正。否则仅仅丢弃而不重传，只有当目的高层协议发现时才进行重传，但这时已经晚了，可能要重传较多数据，对网络资源有些浪费。
  2. 缺点：高层协议是不可靠的传输协议UDP时，并不会带来好处。增加了可靠性，牺牲了实时性。
PPP协议特点
  1. 简单：接收方收到一个帧就进行CRC检验，正确则收下，错误则丢弃。
  2. 封装成帧。
  3. 透明性。
  4. 支持多种网络层协议。
  5. 支持多种链路类型。
  6. 不使用帧的编号，因为PPP不需要实现可靠传输。

Posted on 2020-04-11 Edited on 2020-04-14 In Leetcode题解
寻找两个有序数组的中位数

    给定两个大小为 m 和 n 的有序数组 nums1 和 nums2。请你找出这两个有序数组的中位数，并且要求算法的时间复杂度为 O(log(m + n))。 你可以假设 nums1 和 nums2 不会同时为空。

示例:
    • 输入: nums1 = [1, 3]，nums2 = [2]
    • 输出: 2.0

    • 输入: nums1 = [1, 2]，nums2 = [3, 4]
    • 输出: 2.5

分析
看到复杂度为O(log(m+n))则基本上是二分法了。还需要利用中位数的特点：将这些数切割成左右两部分，当长度相等且left_part<right_part时，切割位置的附近即中位数。

left partright_part
nums1[0],nums1[1], …,nums1[i-1]nums1[i],nums1[i+1],…nums1[m-1]
nums2[0],nums2[1], …,nums2[j-1]nums2[j],nums2[j+1],…nums2[n-1]

  • 可以假设nums1的长度为较小的那个，如果nums2的长度大于nums1，则交换两者。
  • 长度相等，即i+j=m-i+n-j或者i+j=m-i+n-j+1，这是由于有奇偶两种情况，当m+n为奇数时，让左半部分多一个数，最后的中位数则也在左半部分。j由此可以由i表示为j = (m + n + 1) / 2 - i这一个式子，偶数时多加了个1，但是除以2取整相当于没有加。
  • 然后就有三种情况：
    1. 左上大于右下，nums1[i-1]>nums2[j]，注意此时要限制i>left，否则可能超出范围。说明nums1左边大了，应该向左收缩边界，故right=i-1。
    2. 左下大于右上，nums2[j-1]>nums2[i]，注意此时要限制i<right，否则也可能超出范围。说明nums2左边大了，应该向左收缩边界，由于i和j作用是相反的，而j是由i确定的，所以left=i+1。
    3. 左边全部小于右边，这时还需要处理四个边界：i=0,i=m,j=0,j=m。i=0代表nums1左边没有数，则maxleft=nums2[j-1]。同理可以推出其它三种情况。如果是奇数，直接返回左边最大的数。如果是偶数，则返回(maxleft+minright)/2。

无重复字符的最长字串


    给定一个字符串，请你找出其中不含有重复字符的最长子串的长度。

示例:
    • 输入: "abcabcbb"
    • 输出: 3
    • 解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。

    • 输入: "bbbbb"
    • 输出: 1
    • 解释: 因为无重复字符的最长子串是 "b"，所以其长度为 1。

    • 输入: "pwwkew"
    • 输出: 3
    • 解释: 因为无重复字符的最长子串是 "wke"，所以其长度为 3。请注意，你的答案必须是子串的长度,"pwke" 是一个子序列，不是子串。

分析

  • 子串的长度必然与下标有关
  • 要求子串不重复，则考虑到用hash表来建立字符和下标的映射关系。
  • 一旦发现了一个重复的字符，则下标位置如何转移是需要重点考虑的。
  • 结合上面几点，要与下标i+1建立映射而不是i，比如"abca"从a出现重复，这时要从第一个a的下一位开始。

两数相加


    给出两个非空的链表用来表示两个非负的整数。其中，它们各自的位数是按照 逆序 的方式存储的，并且它们的每个节点只能存储一位数字。 如果，我们将这两个数相加起来，则会返回一个新的链表来表示它们的和。 您可以假设除了数字 0 之外，这两个数都不会以 0 开头。

示例:
  • 输入：(2 -> 4 -> 3) + (5 -> 6 -> 4)
  • 输出：7 -> 0 -> 8
  • 原因：342 + 465 = 807

分析

就像加法一样，对两个链表从左向右遍历，并且需要考虑进位。需要注意：

  • 两个链表长度可能不同，当一个链表为空时，需要将其值设为0。
  • 到循环结束时如果carry还是1，需要单独考虑，比如5+5=10，循环结束后结果为0，显然不对。

  • 搜索一个给定的目标值，如果数组中存在这个目标值，则返回它的索引，否则返回 -1 。
  • 你可以假设数组中不存在重复的元素。你的算法时间复杂度必须是O(log n) 级别。

示例

    • 输入: nums = [4,5,6,7,0,1,2], target = 0

  • [6, 7, 1, 2, 3, 4, 5]这种，nums[start] > nums[mid],此时右边是有序的，如果nums[mid] < target ≤ nums[end],则在后面查找，否则去前面。

下一个排列

  • 实现获取下一个排列的函数，算法需要将给定数字序列重新排列成字典序中下一个更大的排列。
  • 如果不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。
  • 必须原地修改，只允许使用额外常数空间。

示例

  • 类似于整数，希望下一个数比当前的数大，因此需要将后面的大数与前面的小数交换。比如123456可以变为123465。
  • 要求下一个数增加的幅度尽可能小，需要：
    • 尽量在靠右的位置进行交换，即从后往前查找。
    • 应该将尽可能小的大数和前面的小数进行交换，如123465,应该将5和4进行交换而不是6和4。
    • 交换后应该将大数后面所有的数按升序排列。以123465为例，交换得到123564，需要将5之后的数升序排列得到123546，显然小于123564。

IP地址和硬件地址的区别

关于子网掩码

  • 255.255.255.0代表的含义？

    • C类地址的子网掩码默认值。
    • A类或 B类地址的子网掩码。

  • 子网掩码为 255.255.255.248，该网络可以连接多少主机？

        248 = 128 + 64 + 32 + 16 + 8，剩余 3位，除去全 0和全 1，剩下 23-2即 6个主机可以连接。

  • A类子网掩码 255.255.0.255是有效的，但是不推荐这样使用。

MTU

  • 是针对数据链路层的。
  • IP数据报的总长度（ 数据 +首部）不能超过这个值。
  • 总长度为 16位，因此有 IP数据报首部的总长度≤min{MTU, 65535} 。

关于分片的一道习题

    一个3200位长的TCP报文传到IP层，加上160位的首部后成为数据报。下面的互联网由两个局域网通过路由器连接起来，但第二个局域网所能传送的最长数据帧中的数据部分只有1200位，因此数据报在路由器必须进行分片。试问第二个局域网向其上层要传送多少比特的数据（这里的“数据”当然指的是局域网看见的数据）？

    重点就是局域网所能传送的最长数据帧中的数据部分这句话所代表的是什么，是IP层还是MAC层？答案是MAC层，因为所有的传输都是由上向下一层一层增加首部最后由MAC层完成的。根据这个倒推上去，得到IP层的数据部分为1040bit。
    TCP给IP的有3200bit，有3200 = 1040 + 1040 + 1040 + 80，所以要分4片，每片都需要加上160bit的首部。

    因此，最终结果为 1200 + 1200 + 1200 + 240 = 3840bit。

路由选择协议

RIP
     RIP是分布式的基于距离向量的路由选择协议，最大的优点是简单。
特点：
  • 仅和相邻路由交换信息。
  • 路由器交换的信息是当前本路由器所知道的全部信息(路由表)。
  • 按固定的时间间隔交换路由信息。

OSPF
     OSPF最主要的特征就是使用分布式的链路协议状态。
特点：
  • 向本自治系统中的所有路由器发送信息(洪泛法)。
  • 发送的信息是与本路由器相邻的所有路由器的链路状态(部分信息)。
  • 只有当链路状态发生变化时，路由器才向所有路由器用洪泛法发送此信息。

BGP
    BGP采用路径向量路由选择协议。
特点：
  • BGP在自治系统之间交换可达性信息。
  • 自治系统之间的路由选择必须考虑有关策略。
  • BGP只是寻找一条能够到达目的网络且比较好的路由，而非最佳。

IGMP协议

IGMP：网际组管理协议。
作用：让连接在本地局域网上的多播路由器知道本局域网上是否有主机参加或退出了某个多播组。
阶段：
  1. 当某个主机加入新的多播组时，该主机应向多播组的多播地址发送一个IGMP报文，声明自己要成为该组的成员。本地的多播路由器收到IGMP报文后，还要利用多播路由选择协议把这种组成员关系转发给互联网上的其他多播路由器。
  2. 组成员关系是动态的。本地多播路由器要周期性地探询本地局域网上的主机，以便知道这些主机是否还继续是组的成员。
隧道技术：
    适用于多播组的位置在地理上很分散的情况。比如说：网1和网2都支持多播，但是网1和网2之间的路由器不支持，则可以对多播数据报进行再次封装，即加上普通数据报首部，通过“隧道”发送。收到后再去掉首部恢复成原来的多播数据报。

VPN和NAT

VPN：虚拟专用网络。
特点：
    采用TCP/IP技术和利用公用的互联网作为通信载体，使一个机构中分布在不同场所的主机能够像使用一个本机构的专用网那样进行通信。
优点价格比建造专用网便宜。
缺点需要比较复杂的技术
分类：
  1. 内联网。
  2. 外联网
  3. 远程接入VPN
NAT：网络地址转换。
NAPT：网络地址与端口号转换，是使用端口号的NAT。
优点可以通过使用NAT路由器使专用网内部的用户和互联网连接。
缺点
  • 通过NAT路由器的通信必须由专用网内的主机发起。
  • 当NAT路由器只有一个全球IP地址时，专用网内最多只有1个主机可以接入互联网。
    • NAPT由于还使用了运输层的端口号，因此在NAPT上的一个全球IP地址，可以供专用网中的多个主机使用。

    Tips

    • 地址数=末地址-首地址，转换为10进制即可。
